Throughout this project, I focused on writing code that is modular, readable, and adaptable—especially evident in the reusable crud.py module created in Project One. By encapsulating MongoDB interactions into well-defined functions, I established a clear separation between data logic and UI components in the dashboard. This structure not only simplified development for Project Two but also made the codebase easier to maintain and extend in the future. The module’s clean, purpose-driven design allowed me to focus on enhancing the user interface and integrating interactive features like dropdown filters, pie charts, and maps without duplicating database logic. This kind of modularization is critical for long-term scalability and reusability across future projects that might involve similar CRUD operations or different datasets.

As a computer scientist, I approached the problem by analyzing user needs, breaking requirements into discrete, testable features, and using tools like Dash and MongoDB to build a responsive, data-driven solution. This project was more client-focused and data-integrated than earlier coursework, requiring careful alignment between backend data queries and frontend user interactions. It showed me the value of choosing the right tools for the job and reinforced the importance of iterative development and testing. Ultimately, computer scientists solve real-world problems through efficient systems and thoughtful design. My work on this dashboard helps Grazioso Salvare visualize and act on animal rescue data more effectively, supporting smarter, faster decision-making and enabling the organization to fulfill its mission more confidently.
